# Feature Specification: 轉輪符號圖庫與速度的集中式設定管理

**Feature Branch**: `001-reel-config-management`

**Created**: 2026-07-07

**Status**: Draft

**Input**: User description: "改版 ReelView 將 SymbolFrames、Speed 統一用 Config、Define 管理；避免新增、調整相關參數會需要將每一個 Reel 掛上的 element 進行調整"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 一處調整速度即套用全部轉輪 (Priority: P1)

維護者想調整所有轉輪的滾動速度，只需在單一設定來源修改速度值，所有轉輪就都採用新速度，
不需逐一打開每個 Reel 節點掛載的元件去改參數。

**Why this priority**: 這是本次改版的核心價值——消除「改一個值要編輯 N 個節點」的重複工，
是使用者提出需求的直接目的；若只實作這一項就已交付主要價值（MVP）。

**Independent Test**: 在設定來源把速度改為某新值，啟動遊戲觀察所有轉輪皆以新速度滾動；
全程未曾編輯任何 Reel 節點上的元件參數，即算通過。

**Acceptance Scenarios**:

1. **Given** 場景中有多個轉輪且皆未各自覆寫速度，**When** 維護者在設定來源修改速度值後啟動遊戲，
   **Then** 每個轉輪都以新的速度滾動。
2. **Given** 維護者只在設定來源改了速度，**When** 檢視各 Reel 節點掛載的元件，
   **Then** 不需要（也沒有）在任何 Reel 元件上手動改動速度即生效。

---

### User Story 2 - 一處增修符號圖庫即套用全部轉輪 (Priority: P1)

維護者想新增、移除或替換轉輪可能出現的符號圖，只需在單一設定來源維護這份符號圖庫，
所有轉輪就都使用更新後的圖庫抽圖，不需逐一在每個 Reel 元件上重設圖片陣列。

**Why this priority**: 符號圖庫是最常需要增修的內容（改版、活動換圖），且目前散落於每個 Reel
元件最痛；與 US1 並列為核心價值。

**Independent Test**: 在設定來源新增一張符號圖後啟動遊戲，觀察該符號會出現在所有轉輪的抽圖結果中；
全程未曾在任何 Reel 元件上編輯圖片陣列，即算通過。

**Acceptance Scenarios**:

1. **Given** 設定來源中的符號圖庫有 N 張圖，**When** 維護者在設定來源加入第 N+1 張圖後啟動遊戲，
   **Then** 所有轉輪滾動時都可能抽出這張新圖。
2. **Given** 維護者只在設定來源調整了圖庫，**When** 檢視各 Reel 節點掛載的元件，
   **Then** 不需要（也沒有）在任何 Reel 元件上手動重設圖片陣列即生效。

---

### User Story 3 - 新增轉輪不需重複填參數 (Priority: P2)

維護者在場景中新增一個轉輪時，該轉輪自動沿用集中設定的符號圖庫與速度，
不需為新轉輪逐項填入圖片陣列與速度。

**Why this priority**: 提升擴充效率並保證新舊轉輪一致；屬於前兩項的延伸效益，非達成核心價值的必要前提。

**Independent Test**: 複製或新增一個乾淨的轉輪節點（未填任何圖庫／速度參數），啟動遊戲後該轉輪
與既有轉輪表現一致（相同速度、相同圖庫來源），即算通過。

**Acceptance Scenarios**:

1. **Given** 已有集中設定，**When** 維護者新增一個未填參數的轉輪並啟動遊戲，
   **Then** 該轉輪以集中設定的速度滾動並使用集中設定的符號圖庫。

---

### Edge Cases

- 當集中設定的符號圖庫為空時，轉輪不得崩潰；應維持既有的「無圖則略過指派」安全行為。
- 當集中設定的速度為 0 或負值時，系統需以既有的預設速度回退，而非讓轉輪停滯或行為異常。
- 現存場景中的轉輪在改版後首次載入時，仍能正常滾動、換圖、停輪對齊，不因設定來源改變而失效。
- 尚未載入或找不到集中設定時，轉輪需以安全預設值運作並可被診斷（不得靜默地表現異常）。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系統 MUST 提供單一集中設定來源，統一保存轉輪的符號圖庫與滾動速度。
- **FR-002**: 所有轉輪 MUST 於執行時從該集中來源取得符號圖庫與速度，作為預設行為。
- **FR-003**: 維護者 MUST 能只在集中來源修改速度一處，即讓所有（未特別覆寫的）轉輪套用新速度，
  無需編輯任何 Reel 節點上的元件。
- **FR-004**: 維護者 MUST 能只在集中來源增修符號圖庫一處，即讓所有轉輪的抽圖使用更新後的圖庫，
  無需編輯任何 Reel 節點上的元件。
- **FR-005**: 新增的轉輪 MUST 在未填任何圖庫／速度參數時，自動沿用集中設定的值。
- **FR-006**: 系統 MUST 在集中設定缺失、圖庫為空或速度非正值時，以安全預設回退並維持不崩潰。
- **FR-007**: 改版 MUST 保留既有轉輪的滾動、換圖與停輪對齊行為（僅改變參數來源，不改變動畫觀感）。
- **FR-008**: 集中設定 MUST 為所有可調參數提供具名、可讀的定義（不得以無說明的魔術數字散落）。

### Key Entities *(include if feature involves data)*

- **符號圖庫 (Symbol Library)**：轉輪滾動時可能抽出的符號圖片集合；為所有轉輪共用的單一清單。
- **轉輪行為設定 (Reel Behavior Settings)**：轉輪的滾動速度及相關可調行為參數，集中定義、具名可讀。
- **集中設定來源 (Central Config Source)**：上述兩者的單一權威來源；轉輪執行時由此取得參數。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 調整全體轉輪速度所需編輯的位置由「每個轉輪各一處」降為「僅設定來源一處」
  （N 個轉輪由 N 處降為 1 處）。
- **SC-002**: 增修符號圖庫所需編輯的位置由「每個轉輪各一處」降為「僅設定來源一處」。
- **SC-003**: 維護者可在 1 分鐘內完成「改速度並讓全部轉輪生效」的操作，且過程零次編輯 Reel 元件。
- **SC-004**: 新增一個轉輪達到與既有轉輪一致表現所需填寫的參數數量為 0。
- **SC-005**: 改版後既有場景的轉輪行為（滾動、換圖、停輪對齊）與改版前一致，無可觀察的退化。

## Assumptions

- 所有轉輪共用同一份符號圖庫與同一速度；本次改版以「集中共用」為單一真實來源。
  是否保留「個別轉輪覆寫」能力見下方待釐清事項。
- 「相關參數」以使用者明確點名的 SymbolFrames（符號圖庫）與 Speed（滾動速度）為核心範圍；
  其餘停輪／減速等純內部調校常數是否一併納入集中設定，見下方待釐清事項。
- 集中設定的維護對象為專案開發者／關卡設計者（非終端玩家）。
- 改版不改變玩法或動畫觀感，純粹是參數來源與管理方式的重構。
